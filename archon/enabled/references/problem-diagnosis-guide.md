# 问题诊断指南 (Problem Diagnosis Guide)

> 使能层 Phase 1 的参考文件 — 从模糊问题到清晰的模型栈

---

## 诊断流程

```
用户问题
  → 提取关键词
    → 匹配问题类型
      → 从场景表中找到最接近的场景
        → 调取推荐模型栈
          → 根据问题复杂度和用户需求调整深度
            → 展示诊断结果并确认
```

## 深度调节机制

使能层支持三个深度级别，用户在 Phase 1 可以选择：

### 🔹 快速诊断 (单模型)
- 时间：2-5 分钟
- 产出：单一模型视角的简要分析
- 适用：用户只需要一个方向，或快速验证假设
- 实现：模型栈只取第一个，Phase 3 跳过

### 🔶 标准分析 (2-3 模型)
- 时间：10-20 分钟
- 产出：多模型合成全景 + 行动建议
- 适用：大多数日常问题
- 实现：取模型栈前 2-3 个，执行全部 5 Phase

### 🔴 深度审查 (4-10 模型)
- 时间：30-90 分钟
- 产出：完整的分析报告/设计文档/长期规划
- 适用：关键决策、系统设计、战略规划、学习体系设计、精英教育规划
- 实现：使用全部推荐模型栈，合成策略为"对立平衡"或"递进深化"
- ⚠️ 设计/规划类任务(学习体系、教育规划等)通常需要 6-10 模型管道；分析诊断类任务 4-5 模型即可

## 诊断调优规则

1. **如果用户的问题跨越多个类别**（比如"团队士气低落导致项目延期"），合并两个类别的模型栈，取交集中的前 4-5 个
2. **如果用户已经引用了某个模型**（比如"用流动模型分析"），从该模型开始，扩展相关模型
3. **如果 52 个场景中没有精确匹配**，用 BFS-DFS(017) 的思路：先广搜（从 6 个类别中找最接近的），再深搜（选 2-3 个最相关的模型）
4. **如果用户的表述极短**（如"这个系统怎么优化"），先进入澄清模式——补充 2-3 个追问再诊断
