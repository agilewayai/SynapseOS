# 🔍 BFS vs DFS 搜索思维 — 模型 017

> **分类**: 🛠️ 方法

## 核心定义

遍历图/树的两种基础策略：**BFS（广度优先搜索）** 逐层遍历，先看所有邻居再深入；**DFS（深度优先搜索）** 沿一条分支走到底再回溯。不是选择"遍历什么"，而是选择"怎么遍历"。

**根本洞察**：BFS 和 DFS 不只是算法，它们是两种最基本的认知策略——一个适合摸清全貌，一个适合深入验证。

## 两种策略对比

| 维度 | BFS（广度优先） | DFS（深度优先） |
|------|---------------|---------------|
| 策略 | 逐层展开，先看全部邻居 | 一条路走到黑，再回溯 |
| 数据结构 | 队列（先进先出） | 栈（后进先出） |
| 空间复杂度 | 高（存所有同层节点） | 低（只存当前路径） |
| 适合场景 | 最短路径、分层探索 | 深度分析、回溯问题 |
| 风险 | 内存溢出（无限广度） | 死胡同（无限深度） |
| 认知模式 | "我想先看看有什么选项" | "我就选这个方向深入看看" |

## 核心原则（7 条）

### 1. BFS 优先原则
如果不确定怎么做，先 BFS 再 DFS。用 20% 的时间做 BFS（扫全貌），再用 80% 的时间 DFS（深钻最有希望的方向）。

### 2. BFS 保证最短路径
如果你需要"找到最快到达目标的路径"——BFS 可以保证最短路径（在所有边权重相等的前提下）。DFS 不能保证。

### 3. DFS 节省内存
如果探索空间很大但深度不大，BFS 可能耗尽内存；如果探索空间很深但广度不大，DFS 更合适。

### 4. 需要回溯时用 DFS
如果问题需要"尝试一种可能→发现不行→回到上一个选择点→尝试另一种可能"——DFS 是天然匹配（拓扑排序、八皇后、解数独）。

### 5. 需要全貌时用 BFS
如果你需要了解一个系统的"所有可能性"——BFS 确保你不会遗漏任何一个分支。层层推进，没有分支会被跳过。

### 6. 混合策略往往最优
大多数现实问题需要混合策略。先 BFS 扫 2-3 层建立地图，再 DFS 深入最有希望的路径——然后回到根节点 BFS 到其他分支。

### 7. 切换信号比坚持更重要
DFS 虽然强调深度，但知道"什么时候该切换回 BFS"才是最关键的技能。如果在 DFS 中连续多次没有进展——这是一个强烈的切换信号。

## 实践要点

- **BFS 建立心智地图**：进入新领域时，先不要深钻任何一个方向——先画出一张"这个领域有哪些方向"的地图
- **DFS 要有时间盒**：深钻需要专注，但也需要设限——"在这条路径上探索 2 小时，如果没有突破就回溯"
- **根节点选择决定一切**：BFS 和 DFS 的效率高度依赖你从哪个节点开始探索。选错入口，两种策略都很低效
- **反模式：永远 BFS（调研过度）**——花了 90% 的时间调研各种可能性，但从未深入任何一个
- **反模式：永远 DFS（钻牛角尖）**——在一个方向上走到黑，即使信号已经明确方向错了
- **反模式：把 BFS/DFS 当人格标签**——"我是 BFS 型思考者"——最好的思考者是根据问题自如切换两者的人
- **启发**：面对新问题时，先问"我现在应该 BFS 还是 DFS？"

## 典型案例（8 个）

### 1. 求职策略（BFS vs DFS）
- BFS：投 30 家公司的简历，每家做初步了解，看看哪个方向机会最多
- DFS：锁定 3 家最想去的公司，每家公司深入研究，定向准备面试
- 最佳策略：先 BFS 投 30 家感受市场（了解行业在招什么方向），然后选 3 家 DFS 深入准备

### 2. Debug 的 BFS-DFS 切换
- BFS：先检查错误范围——是全局的（所有用户都挂了）还是局部的（某用户、某模块、某时段）
- 确定了"这个 API 有问题"（BFS 定位到了范围）→ DFS：深入追踪这个 API 的调用链，直到找到根因
- 大部分初级工程师的问题是：跳过了 BFS，直接 DFS——在一个高概率无关的方向上 Deep Dive

### 3. 学一门新编程语言
- BFS：先花 1 天快速了解语法、类型系统、工具链、社区——画出概念地图
- DFS：选择你最需要的方向深入（比如 Web 开发）——写一个小项目
- 没有 BFS 的 DFS 学习：卡在细节中，不知道为什么学这个
- 没有 DFS 的 BFS 学习：读了很多教程但写不出可用的程序

### 4. Google 的 PageRank 算法
PageRank 本质上是 BFS 的一个变体——它从每个网页出发，沿链接向外探索，统计"被最多链接指向"的页面。BFS 确保覆盖了全网的链接结构。

### 5. 产品路线图的探索
- BFS 阶段：了解所有可能的方向——用户需求、市场趋势、技术可能
- DFS 阶段：选 1-2 个核心方向深入验证——做原型、做用户测试
- 大多数产品的失败是把 DFS（深度）做在了错误的方向上——BFS 选错了根节点

### 6. 犯罪调查
K BFS：收集所有线索和证据（不再排除任何可能性之前先全面了解）
- DFS：锁定一个嫌疑人，深入调查他的不在场证明、动机、机会
- BFS-DFS 切换：如果深入调查的嫌疑人被证明无罪（DFS 走到死胡同），回到 BFS 重新选择下一个方向

### 7. 知识体系构建
- BFS：先了解一个学科的"知识地图"——有哪几个子领域、核心概念是什么
- DFS：每个子领域选 1-2 个核心概念深钻，建立深层理解
- 最佳：BFS(学科全貌)→DFS(概念1)→DFS(概念2)→BFS(回到全貌补充)→DFS(概念3)

### 8. Reddit/论坛信息探索（Scrolling 模式）
- BFS：浏览首页——看多类话题，不深入任何一个
- DFS：点进一个子版块看全部内容
- 为什么你有时候觉得刷 Reddit 很累？因为 BFS 和 DFS 混在一起当了——你既在扫首页又在一个帖子里深钻，认知负荷叠加

## 💡 实战练习

1. **BFS 先扫**：面对新问题时，先花 20% 时间列出所有可能方向（不深入任何一个）。
2. **DFS 时间盒**：在锁定方向后，设定"2 小时内只深入这一个方向，期间不看其他选项"。
3. **切换信号**：下次深钻没有进展时，问"我需要回到 BFS 重新选择方向吗？"
