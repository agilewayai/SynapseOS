# 🩺 RICE 全科问诊 — 模型 024

> **版本**: 1.0.0 | **作者**: Arthur

## 核心定义

全科医学临床诊断的底层思维模型——通过 **R**ed Flags（危险信号排除）、**I**nterview（系统性病史采集）、**C**linical Exam（精准体格检查）、**E**vidence Synthesis（诊断证据综合）四步闭环，在不确定性中做出最佳临床决策。根本洞察：**诊断不是找到"正确答案"，而是在有限的、不完全的信息中排除最危险的选项，再逐步缩小可能性。**

## 四步诊断流程

### 🚩 R — Red Flags：排除致命/急症/不可逆风险
核心：先做排除法——排除最坏情况。
- **临床**：胸痛患者先排除 ACS（心梗）→气胸→PE（肺栓塞）
- **系统**：服务不可用时先排除硬盘满→网络中断→数据库宕机
- **项目**：项目延期时先排除"核心开发人员要离职"→"技术上做不到该需求"
- **特征**：这些情况的共同点：概率低但致命——你不能错过它们

### 🗣️ I — Interview：系统性病史采集
核心：从"主诉"开始，系统性地向"可能的诊断方向"收敛。
- **开放式问题**：说清楚发生了什么？
- **聚焦式问题**：什么时候开始的？怎么样触发？频率如何？之前遇到过吗？
- **5 个标准提问（全科医学通用）**：
  1. 什么时候开始的？
  2. 怎么开始的？/ 做了什么之后开始出问题？
  3. 有什么让情况变好/变坏的因素？
  4. 你身体/系统其他地方有没有什么问题？
  5. 你之前有没有发生过类似情况 / 有过类似操作？

### 🔬 C — Clinical Exam：精准体格检查
核心：用检查验证鉴别诊断方向。
- **临床**：听诊（肺音/心音）+ 触诊（压痛/肿块）+ 望诊
- **系统**：查看日志文件、网络延迟测试、CPU/内存/IO 状态、链接检查
- **特征**：C 的输入是 R+I 产生的**假设**。不是随意检查，而是**假设驱动的检查**。

### 🧠 E — Evidence Synthesis：综合诊断
核心：将所有线索整合 → 概率评估 → 决策。
- **整合所有信息**：R 排除最坏+ I 病史+C 检查 → 画出全貌
- **概率评估**：
  - "这个病人在当前数据下的最可能诊断是什么？"
  - "排除最坏情况后，剩下的可能性按概率排序"
- **贝叶斯更新**：每个 I/C 新发现都是一个"证据" → 修正对各个可能性的概率估计
- **决策分岔**：
  - 确诊：启动治疗
  - 无法确诊但排除了危险：观察等待(用时间做诊断)
  - 无法确诊且仍有危险：约转诊/进一步检查

## 核心原则（7 条）

### 1. R 优先——先排除"不能错过的"
R 必须优先于 IC。**原则**："先排除致命的+紧急的+有不可逆后果的风险，再慢慢找最可能的原因。"不先排除危险就去 I+C，R 的漏诊后果是灾难性的。

### 2. 概率思维——不是"是/否"而是"可能性有多高"
**原则**：诊断不是二分法（对或错），而是概率连续体（最可能是……但也不能排除……）。用一个"可能性分布"替代"一个诊断"。

### 3. 检验分层——无创→有创
**原则**：选择检验的顺序应遵循：从最简单、最安全、最便宜的到最复杂、最冒险、最昂贵的。"先量血压"不是一个检查技巧——而是一个原则问题。

### 4. 用时间做诊断
**原则**：当没有紧急危险时，"观察+随访"本身就是一个有效的诊断工具。"等 24 小时"可能比昂贵的检查更有诊断价值。

### 5. 贝叶斯更新——每个新证据修正概率
**原则**：每一次检查都是一个贝叶斯更新的机会。检查前与检查后的"概率差值"决定了检查的价值。

### 6. 明确退出条件
**原则**：定义"什么时候可以停止思考、给出结论"。输出三个条件之一：确诊、安全可观察、需转诊。

### 7. "正常的阴性"同样有价值
**原则**："排除了一个重要的可能性"本身就是诊断进展。不是只有"找到正确答案"才是进步——排除法走完三个大方向仍然无解，这叫"排除了可能的重大疾病"，也是进展。

## 实践要点

### 鉴别诊断列表（DDx）的构建
鉴别诊断列表是框架的核心：列出所有合理的可能诊断，并按其概率/风险排序。全科医生常用的"五个标准提问"本质上是自动构建 DDx。

### ⚠️ 反模式

| 反模式 | 表现 | 解药 |
|--------|------|------|
| 跳过 R | "不可能是那种情况"→没检查就排除了 | 将 R 检验标准化为强制性检查清单 |
| 确认偏误 | I 阶段只听支持自己假定诊断的信息 | I 阶段先开放式问题，再聚焦；E 阶段故意找反面证据 |
| 过早关闭 | Interview 还没做完就跳到结论 | 强制要求至少列 3 个鉴别诊断再进入 C |
| "技术性"玩偶综合征 | 过度依赖检查而非问诊 | 记住：I(问诊) + C(检查) 的诊断贡献大约各占60%和20% |

## 典型案例

### 1. 胸痛急诊
- **主诉**：41 岁男性、胸痛 2 小时、从公司直接来的
- **R**：先排除 ACS（心梗等高危）→ 心电图正常 → 血压正常 → 肌钙蛋白正常 → **排除心梗**
- **I**：问诊发现"最近工作中压力很大、熬夜多、偶尔胸痛、深呼吸时加重、躺平缓解"
- **C**：触诊胸骨有压痛、局部皮肤无红肿 → 大概率是**肋软骨炎**
- **E**：整合后→排除 ACS（R）→ 病史支持肋软骨炎（I+C）→ 结果：急性肋软骨炎 → 不开抗焦虑药、用外用消炎药 + 建议调整工作节奏 → 2 周后随访已好转

### 2. 线上服务不可用 — DevOps 排障
- **告警**：API 超时率从 0.1% 飙升到 30%
- **R**：先检查数据库是否完好 → 数据库正常；检查是否被攻击 → 没有 DDoS→ **排除最危险的两个选项**
- **I**：查看最近 15 分钟的发布记录、流量变化、上游依赖状态 → 发现 2 小时前有数据库连接池配置变更
- **C**：查看连接池指标 → 实际连接数在飙升、空闲连接池配置参数从 50 调整到了 10
- **E**：新配置引起连接饥饿 → 紧急回滚配置 → 服务恢复 → 根因分析

### 3. 慢性病鉴别诊断
- **主诉**：65 岁男性、背痛 3 个月
- **R**：无发热、无夜间痛、无神经系统症状 → **排除感染/肿瘤/脊髓压迫**
- **I**：隐痛、弯腰时加重、自行服布洛芬效果一般
- **C**：腰椎活动度正常、直腿抬高试验阴性
- **E**：列出鉴别诊断 → 骨关节炎(高) ≥ 强直性脊柱炎(中低) > 骨质疏松(不定) → 建议 X 光并安排骨密度检查
- **结果**：X 光显示轻度骨关节炎 + 骨密度 T 值 -2.5 = 骨质疏松 → 治疗方案兼治两者 + 6 个月随访

### 4. 投资决策的 RICE 映射
- **R**：先排除"本金全部损失"的可能性（核实公司财务报表）→ 看看有没有债务到期压力的信号
- **I**：问 CEO 和团队背景、整体营收趋势、竞争格局——用开放式问题
- **C**：看核心数据：月活跃、毛利率、现金流周期
- **E**：综合判断——"大概率这个投资是合理的，但要定期检查现金流指标"

## 跨模型关联

### 推荐场景组合
> **技术/产品风险评估**: 024 + 018 + 010 + 020
> **故障排查/事故响应**: 024 + 003 + 017 + 005
> **业务连续性/灾备规划**: 024 + 025 + 004 + 023
> **个人焦虑/决策困难**: 024 + 019 + 016 + 005

- **RICE(024) + BFS-DFS(017)**：**最自然的一对组合**——R = 先 BFS 扫一遍所有可能的 Red Flags（广度优先的排除），I+C+E = 目标明确的 DFS 深入验证（深度优先的确认）。
- **RICE(024) + 状态机(003)**：人体/系统本质上都是一个状态机。RICE 问诊的核心就是判断当前处于哪个状态（正常？危险？失代偿？）。
- **RICE(024) + 双系统(016)**：经验丰富的老医生走大量的系统 1（模式识别），但每到关键的鉴别诊断节点，需要强制切到系统 2 的慢思考。RICE 框架本身就是系统 2 的外骨骼。
- **RICE(024) + 元认知(019)**：好医生的元认知体现在："我现在对这个诊断的信心是否合理？我排除了足够多的危险选项吗？我是否因为昨天见过类似病例而过度锚定？"
- **RICE(024) + 最小阻力(009)**：**致命的陷阱**——按最小阻力路径走：最常见的病 → 开最常见的药 → 错过了不常见但致命的原因。RICE 的 R 步骤就是专门对抗最小阻力的"安全检查"。
- **RICE(024) + 逆向(005)**：逆向思维在 RICE 中的应用："如果我的诊断是错的，最可能的正确诊断是哪个？我排除了它吗？"

## 💡 实战练习

### 初级
初级：医学RICE的核心是"先排危险再找原因"。对一个你正在诊断的问题，先列出所有"可能造成严重后果"的原因，排除后再考虑轻微原因。

### 中级
用RICE框架对团队的产品Backlog排序：Reach(影响面)、Impact(影响力)、Confidence(确定度)、Effort(成本)。计算每个项的RICE得分，按分数排序。原顺序和新顺序的差异有多大？

### 高级
为你的季度规划引入RICE排序。关键陷阱：Impact和Confidence经常被混淆——高Impact但低Confidence的项，你如何处理？

### 反思
RICE教你的是"所有的优先级排序都包含主观判断"。Confidence这个维度就是承认这一点。你上次说"这是最重要的"时，你的Confidence是多少？你验证过吗？
