# 🎙️ Archon 采访协议（Interview Protocol）

> **版本**: 1.0.0 | **定位**: 使能层之前的问题发现与校准引擎

## 定位

采访协议是 Archon 的最前端校准入口——在启用模型管道与动作能力之前，先用结构化对谈把**真正的问题**从**最初的陈述**中提炼出来。

```
Archon 校准入口        →  发现问题 + 校准问题 + 收集全维度信息
  ↓ Interview Summary（结构化采访纪要）
Archon 执行引擎        →  模型选择 + 管道执行 + 动作调用 + 合成产出
  ↓
Xuan Master 核心       →  27 个深层认知模型
```

### 为什么需要采访层？

| 没有采访层 | 有采访层 |
|-----------|---------|
| 用户说"我想减肥" → 直接出减肥计划 | 用户说"我想减肥" → 采访发现真正问题是"体检报告血脂偏高，医生建议减重" → 计划聚焦健康而非体重 |
| 用户说"帮我分析职业转型" → 假设用户想转行 | 采访发现用户其实热爱当前领域，只是需要换环境而非换赛道 |
| 分析建立在模糊假设上 | 分析建立在经过校准的事实上 |

## 采访协议

### 三阶段节奏

```
Round 1: 广度扫描 (BFS)
  目标：覆盖问题的所有维度，不遗漏任何关键信息
  驱动模型：014 5W2H + 017 BFS
  时间：4-7 个问题
  输出：问题的"全貌地图"

Round 2: 深度钻取 (DFS)
  目标：在 Round 1 发现的最关键维度上深挖
  驱动模型：006 第一性原理 + 005 逆向思维
  时间：3-5 个追问
  输出：隐藏假设 + 真实约束 + 成功/失败画面

Round 3: 校准确认
  目标：确认 Agent 理解的"真实问题"和用户想解决的问题是一致的
  驱动模型：019 元认知 + 022 麦肯锡
  时间：1-2 个确认性问题
  输出：精炼后的问题陈述（Interview Summary）
```

### 采访工具箱：四类问题

| 类别 | 驱动模型 | 典型问题 | 什么情况下用 |
|------|---------|---------|------------|
| **结构扫描** | 014 5W2H | "这是为谁做的？时间约束是什么？资源边界在哪？" | 任何采访的第一轮——先建立全局框架 |
| **逆向校验** | 005 逆向思维 | "如果这件事完全失败了，最可能的原因是什么？" / "你希望一年后回头看，最好的结果是什么样？" | 当用户对目标描述模糊时——用终局倒逼清晰度 |
| **本质追问** | 006 第一性原理 | "抛开所有的'应该'和'别人说'——你内心深处真正想要的是什么？" / "如果没有任何限制，你会怎么做？" | 当用户陈述中包含大量"社会期待"或"别人说"时 |
| **假设揭示** | 019 元认知 | "你刚才说'我只能这样'——这个'只能'是你的判断还是事实？" / "如果你最好的朋友跟你面临同样的问题，你会给他什么建议？" | 当用户表达中有"我必须""我只能""没办法"等封闭性语言时 |

### 采访纪律

| 纪律 | 说明 |
|------|------|
| **不问连续引导性问题** | "你觉得是不是因为X？"——这种问题在替用户回答。改为"你觉得主要原因是什么？" |
| **每次只问一个问题** | 不堆砌三个问题让用户选择回答哪个——一个一个问题来 |
| **追问"为什么"不超过三次** | 三次之后如果用户说"我也不知道"——停下来。标记为"待探索"而非强行追问 |
| **接受沉默** | 用户停顿 5-10 秒不是在"拒绝回答"——是在"思考"。不要填补沉默 |
| **不预设答案** | Round 1 不做任何判断。听到什么就是什么——判断放在 Round 3 |

## 采访流程模板

### Round 1: 广度扫描（5W2H 驱动）

**开场**：
> "为了帮到你，我需要先了解全局。我会问几个问题——不需要每个都有完美答案，想到什么说什么。"

| # | 维度 | 问题模板 |
|---|------|---------|
| 1 | **Why** | "你能多说一点背景吗？是什么让你现在特别想解决这个问题？" |
| 2 | **Who** | "这件事涉及到哪些人？谁会受影响？谁的意见很重要？" |
| 3 | **What** | "你心目中'解决好了'具体是什么样？有没有一个画面或者一个数字？" |
| 4 | **When** | "有没有时间上的压力？什么时间点是必须在此之前完成的？" |
| 5 | **Where** | "这个问题在什么场景下出现？有没有特定的环境或条件？" |
| 6 | **How Much** | "你愿意/能够投入什么？时间？钱？精力？有什么硬约束？" |

**Round 1 结束判断**：当 Agent 感觉自己能画出问题的"基本轮廓"时，进入 Round 2。

### Round 2: 深度钻取（逆向 + 第一性原理驱动）

**从 Round 1 的答案中选出最模糊、最重要或最矛盾的 1-2 个维度深挖。**

| 如果 Round 1 发现... | 追问 |
|---------------------|------|
| 目标描述模糊 | "你刚才说'做好'——如果一年后你在跟朋友聊这件事，你最想说出哪句话来证明'做好了'？" |
| 约束表达绝对化 | "你说'没办法做X'——如果这个约束突然消失了，你第一件事会做什么？" |
| 涉及多个人但优先级不明 | "如果只能让一个人满意，你会让谁满意？为什么是他/她？" |
| 情绪信号强（焦虑/恐惧/无奈） | "在这个问题上，你最怕发生的事情是什么？" |
| 提到"以前试过但失败了" | "上次尝试的时候，你觉得最关键的那个'卡住的点'是什么？" |

### Round 3: 校准确认

**Agent 用自己的话重新表述"真正的问题"，让用户确认。**

> "让我确认一下我的理解是否准确——
>
> 你真正想解决的**不是** [最初的陈述]，
> 而是 **[通过采访发现的更深层/更精确的问题]**。
>
> 具体来说：
> - 核心约束是 [从Round1提取的约束]
> - 成功的标准是 [从Round2提取的成功画面]
> - 最担心的是 [逆向问题揭示的]
>
> 这三条对不对？有没有我理解歪了的地方？"

## Interview Summary 输出格式

采访结束后，生成一份结构化的 Interview Summary，作为后续使能层的输入。

```markdown
# Interview Summary: [问题简述]

## 1. 初始陈述 vs 校准后的问题
- **用户最初说的**：[一句话]
- **采访后发现真正的问题是**：[一句话]
- **关键差异**：[解释为什么两者不同]

## 2. 五维快照
| 维度 | 内容 |
|------|------|
| Why | [为什么现在/为什么重要] |
| Who | [关键干系人+优先级] |
| What Done | [成功的具体画面] |
| When | [时间约束] |
| Constraints | [硬约束+软约束] |

## 3. 发现的隐藏假设
- [假设1] → [采访中怎么发现的]
- [假设2] → [采访中怎么发现的]

## 4. 风险预警
- [逆向分析发现的最大风险]

## 5. 推荐使能层模式
- **深度**: quick / standard / deep
- **模式**: parallel / pipeline
- **产出模板**: analysis / design / decision / retro / cognitive
- **推荐模型栈**: [具体模型编号]
```

## 接入使能层

Interview Summary 生成后，自动传递给使能层：

```
🎙️ Interview → Interview Summary
                  ↓
⚡ Enabled → Phase 1: 基于 Interview Summary 做诊断
                  ↓
            Phase 2-5: 管道执行
```

Agent 在加载 `archon/SKILL.md` 或 `enabled/SKILL.md` 后，Phase 1 应该：
1. 首先检查是否有 Interview Summary 可用
2. 如果有：基于 Interview Summary 直接进入模型选择
3. 如果没有：按照原有流程（简要诊断 + Policy/Mechanism 确认）

## 采访层自动触发条件

Agent 应在以下情况自动触发采访层：

| 触发条件 | 示例 |
|---------|------|
| 问题陈述过于简短（<20 字） | "帮我分析一下职业发展" |
| 问题陈述中包含模糊词 | "搞好一点""想办法""不知道怎么办" |
| 问题涉及多人但未说明优先级 | "团队效率低""家庭关系紧张" |
| 用户表达中有强烈的情绪但未说明原因 | "真的很焦虑""快要崩溃了" |
| 问题陈述中有明显的"社会期待"痕迹 | "我应该是要做..." "别人都说..." |
